# 프로젝트 개요

아마추어 혼성 풋살팀 OTP FC 운영 관리 웹앱 "모이다"

기술 스택

* React 18
* Firebase Firestore
* Firebase Anonymous Auth
* Tailwind CSS
* Leaflet
* PWA
* GitHub Pages

Firebase Project ID

moida-otpfc

기본 컬렉션 경로

artifacts/moida-otpfc/public/data/

---

# 페이지 구성

## 관리자 페이지

* index.html : 홈
* attendance.html : 출석 관리
* roster.html : 회원 / 회비 관리
* team-maker.html : 팀 편성
* match.html : 매치 관리

## 회원 페이지

* member.html : 회원 전용 기능

# 최우선 원칙

기존 기능을 깨지 않는다.

새 기능보다 기존 기능 안정성을 우선한다.

추측으로 구현하지 않는다.

기능 변경 가능성이 있으면 먼저 설명한다.

---

# 비개발자 대응 원칙

사용자는 코드를 작성하지 않는다.

설명은 반드시 기능 중심으로 한다.

좋은 예

* 출석 기능
* QR 기능
* 팀 편성 기능
* 회비 기능
* 회원 관리 기능

좋지 않은 예

* 함수
* 변수
* 훅
* 컴포넌트
* 의존성

사용자에게는 기능 기준으로 설명한다.

# 작업 범위 최소화

매우 중요

사용자가 요청한 기능을 우선 분석한다.

의존 관계가 명확히 필요한 경우에만 최소 범위로 추가 분석한다.
