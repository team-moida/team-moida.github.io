# moida 디자인 기준안 (상세)

> 용도: 새 화면·박스·컴포넌트를 만들 때의 디자인 기준. CLAUDE.md "디자인 기준(요약)"의 상세판.
> 원칙: 값은 고정해서 쓰고 임의 변형을 만들지 않는다. 애매하면 이 문서를 따른다.
> Version 1.0 · 2026-06-23

---

## 1) 색 (Color)

> 현재 적용색 = 이 표(현행 코드 기준). 미래 지향색은 8) 인디고 톤 참고.

각 역할은 **1값으로 고정**. 새 변형(밝게/어둡게) 만들지 말 것.

| 역할 | 값 | 쓰임 |
|---|---|---|
| 강조(Primary) | `#14b8a6` 민트/teal (텍스트·액티브 `#0d9488`) | 주요 버튼·탭 선택·핵심 강조 |
| 보조/포인트 | (현재 별도 보조색 거의 없음) | 핑크는 여성 배지 등 소량 — 거의 미사용, 미래 톤에서 인디고로 대체 예정(8 참고) |
| 글자-진함(Ink) | `#0f172a` (slate-900) | 제목·본문 핵심 |
| 글자-중간 | `#475569` | 본문 보조 |
| 글자-옅음 | `#94A3B8` | 캡션·비활성·플레이스홀더 |
| 배경 | `#F8FAFC` 화면 / `#FFFFFF` 카드 | 페이지/카드 면 |
| 테두리 | `#E2E8F0` | 카드·구분선 |
| 성공 | `#10B981` | 완료·정상 |
| 경고 | `#F59E0B` | 지각·주의 |
| 위험 | `#EF4444` | 노쇼·삭제·오류 |

- **다크모드 1:1 대응**: 라이트의 각 역할에 다크값을 하나씩 매핑(배경↔어두운 면, 글자 명암 반전). 강조·보조·상태색은 **동일 hue 유지**(밝기만 조정), 새 색을 끌어오지 않는다.
- 핑크·민트는 같은 화면에서 **둘 다 강하게 쓰지 않는다**(핑크=주, 민트=보조).

## 2) 타이포 (Typography)

| 단계 | 크기(px) | 무게 |
|---|---|---|
| 페이지 제목 | 22~24 | 800~900 |
| 섹션 제목 | 17~18 | 800 |
| 본문 | 15~16 | 600~700 |
| 보조 | 13 | 600~700 |
| 캡션 | 11~12 | 700~900 |

- **가독성 하한**: 본문 **15px↑**, 캡션 **12px↑**. 글자/배경 **명도대비 4.5:1↑**.
- 11px 이하는 배지·라벨 등 짧은 것만. 길게 읽는 문장에 쓰지 않는다.

## 3) 간격 (Spacing — 8pt 그리드)

- 기본 단위 **8의 배수**(8·16·24·32). 미세조정에만 4 사용.
- 권장값: **카드 안쪽 패딩 16** / 카드 사이 간격 **12~16** / **화면 좌우 여백 16**.
- 묶음 내부 줄 간격은 `gap`으로(개별 margin 남발 금지).

## 4) 모서리·그림자 (Radius · Shadow)

| 요소 | 모서리 |
|---|---|
| 작은 요소(버튼·입력·작은 배지) | `rounded-xl` (12~14) |
| 카드·패널 | `rounded-2xl` (16~20) |
| 알약·토글·동그란 배지 | `rounded-full` |

- `rounded-3xl` 남발 금지(과한 둥글기).
- **그림자는 카드용 1종만**: `0 4px 14px rgba(0,0,0,.05)`. 그 외 추가/변형 금지. (강조는 색·테두리로)

## 5) 아이콘 (Icon)

- **member-icons.js 외곽선 세트 하나로 통일**. 다른 아이콘 파일/이모지 혼용 금지.
- 표준 size: **16 / 20 / 24 / 30** 중에서만. (16=인라인·캡션, 20=기본, 24=섹션, 30=하단 탭)
- 새 아이콘 필요 시 member-icons.js에 같은 외곽선 스타일(stroke=ICON_STROKE)로 추가.

## 6) 정렬 · 컴포넌트 패턴

- **정렬**: "세로 중앙은 기본, 가로 정렬은 보존" — CLAUDE.md "박스 내부 정렬 규칙" 따름.
  여러 줄/다른 글씨는 하나의 묶음으로 묶어 그 묶음을 세로 중앙에 둔다.
- **컴포넌트 공통화 방향(원칙 — 아직 구현은 아님)**: 아래는 반복되므로 점차 공통 컴포넌트로 수렴시킨다.
  - **빈 상태(EmptyState)**: 아이콘 + 제목 + (선택)부가문구, 가로 중앙.
  - **카드(Card)**: 흰 면 + `rounded-2xl` + 카드 그림자 1종 + 패딩 16.
  - **버튼(Button)**: 주(핑크 채움) / 보조(민트·연한 면) / 위험(빨강), `rounded-xl`.
  - **배지(Badge)**: `rounded-full`, 상태색 연한 배경 + 진한 글자.
  - 새로 만들 땐 위 패턴과 색/간격/모서리 기준을 그대로 적용한다.

## 7) 조끼(팀 배지) 색 — 고정 (테마/토큰 변경 제외)
- 팀 배지 A~F 색(getTeamBadge / team-utils.js 등에 정의)은 실제 경기 조끼와 매칭되는
  "기능색"이다. 색 통일·디자인 토큰화·플랫폼 톤 변경 대상에서 **제외**하고 기존 값을 유지한다.
- 이유: 현장에서 조끼 색으로 팀을 식별하므로, UI 테마가 바뀌어도 조끼 색은 그대로여야 한다.
- UI 크롬 색(버튼·탭·헤더·강조)과 조끼 색은 별개 체계로 본다. 둘을 한 토큰으로 묶지 않는다.

## 8) 미래 플랫폼 지향 톤 (참고 — 아직 미적용)
> 여러 종목·익명 팀을 포괄하는 플랫폼으로 확장할 때를 대비한 방향 메모. 현재 앱(핑크/민트)에는
> 아직 적용하지 않는다. 3단계 토큰화가 끝난 뒤 값만 교체하는 방식으로 전환 예정.

| 역할 | 값 | 비고 |
|---|---|---|
| 메인(Primary) | `#183FB0` | 진한 일렉트릭 블루 — 버튼·탭·강조 |
| 메인-딥 | `#122E78` | 눌림(active)·큰 배경 면 |
| 포인트(Accent) | `#C2F94A` | 라임 — 에너지 강조에만 소량(LIVE 등) |
| 베이스/텍스트 | `#15171E` / `#5A5F6E` / `#9AA0AE` | 잉크·본문·옅음 |
| 배경/카드/테두리 | `#F4F5F8` / `#FFFFFF` / `#E6E8EF` | |
| 성공·경고·위험 | `#18C07A` / `#FFB020` / `#FF4D4F` | |

- 톤 방향: "잉크/화이트 골격 + 진한 블루 메인 + 라임 포인트" = 젊고 활기차되 세련됨.
- 폰트 지향: 한글 Pretendard 계열, 무게 대비 크게(굵은 제목 + 가벼운 본문), 숫자는 크고 굵게.
- ★ 위 톤에도 7) 조끼 색 고정 원칙은 그대로 적용된다(조끼는 이 팔레트와 무관).

## 9) 토큰 사용 주의 (다크모드)
- 토큰은 두 종류다:
  · theme-invariant(다크에서도 값 동일): --c-accent, --c-accent-deep, --c-sub,
    --c-success, --c-warn, --c-danger → 어디든 안전하게 사용.
  · theme-varying(다크에서 값이 바뀜): --c-ink, --c-text, --c-bg, --c-card, --c-border
    → html.dark에서 자동 반전됨.
- ★ theme-varying 토큰은 "다크모드여도 라이트로 고정인 영역"(예: 흰 카드 강제 화면)에는
  쓰지 말 것. 자동 반전 때문에 흰 배경에 밝은 글자가 되어 안 보일 수 있다.
  그런 자리엔 기존 고정 hex를 그대로 둔다.
- ★ darkMode 변수로 색을 분기하는 삼항(darkMode?A:B) 안의 hex도 토큰으로 바꾸지 말 것.
  코드 분기 + 토큰 자동반전이 이중 적용되어 색이 뒤집힌다.
- ${color}59 같은 hex-알파 문자열 결합에 쓰이는 색값도 토큰 금지(var()가 깨짐).

## 10) teal 리테마 좌표 (미래 인디고 전환)
- teal 색을 바꿀 땐 두 곳을 함께 고친다:
  ① tailwind-config.js의 teal 값(라이트 팔레트),
  ② common.css의 html.dark .text-teal-* / .bg-teal-* / .border-teal-* 다크 보정.
  한쪽만 고치면 라이트/다크가 어긋난다.
- Tailwind 클래스(bg-teal-500 등)는 var()로 못 바꾸므로, 이렇게 tailwind-config.js의
  팔레트 값만 교체하는 방식으로 전환한다. config는 Tailwind CDN 로드 '다음'에 와야 한다.
