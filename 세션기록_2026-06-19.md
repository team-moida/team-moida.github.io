## 2. 네이티브 푸시 앱 — 까만 화면 점등 성공 ✅
- **목표**: 카톡처럼 꺼진(까만) 화면이 켜지며 알림이 뜨게. 웹 푸시로는 불가 → 앱(TWA)에 푸시만 네이티브로 이식.
- **TWA 소스**: `D:\tmp\moida-twa\source` (PWABuilder, applicationId `app.team_moida`)
- **결과물**: `D:\tmp\모이다.apk` (현재 v1.0.3). 이 파일 하나로만 덮어쓴다.
- **핵심 구현**:
  - IMPORTANCE_HIGH 알림 채널(`moida_push`) — 앱이 자동 생성 → 회원이 설정 안 건드려도 고급 알림.
  - `MoidaFCMService`: 네이티브 FCM 수신 → 그 채널로 표시 + **wake lock으로 화면 강제 점등** + 전체화면 인텐트.
  - `WakeActivity`: 화면 켠 뒤 **지연 종료**(즉시 finish하면 점등 안 되던 버그였음 — 이게 마지막 범인).
  - 권한 USE_FULL_SCREEN_INTENT/WAKE_LOCK, targetSdk 33(권한 자동허용 목적).
  - 웹: index.html에서 nativeToken 캡처+즉시 저장(TWA가 index를 먼저 열어 토큰 유실하던 것 수정). 서버는 nativeToken 있으면 네이티브 우선 발송.
- **검증**: `firebase functions:log` 에서 `[FCM native]` 확인. 실기기(삼성)에서 까만 화면 점등 성공.

## 향후 로드맵 (오늘 논의, 미결정)

### A. 플레이스토어 정식 출시 시 바꿔야 할 것
- targetSdk를 최신(34/35)으로 올려야 함 → 그러면 **USE_FULL_SCREEN_INTENT(까만화면 점등)는 전화·알람 앱만 허용**되어 풋살앱은 거부 위험. **점등 기능은 팀용 사이드로드에만 남기고, 플레이 버전은 일반 고중요도 알림으로** 가는 게 현실적.
- Play 앱 서명 등록 → assetlinks에 구글 서명키 지문 추가. AAB 제출. 서명키 백업 필수.
- 서류: 개인정보처리방침 URL, 데이터 안전(위치=GPS 체크인 주의), 스토어 등록 정보.
- 신규 개인 계정은 **테스터 12명·14일 비공개 테스트** 후 정식 출시 가능. 등록비 $25.

### B. 안드로이드 앱 + 아이폰 웹 동시 운영 — 연동 자동
- 둘 다 같은 Firebase(moida-otpfc) + 같은 웹 코드를 봄 → **데이터 자동 동기화**. 추가 연동 개발 불필요.
- 지금 TWA도 이미 그 구조. Play 출시는 "안드로이드 배포 방법"만 바뀌는 것.
- 알림: 안드=네이티브 푸시 / iOS=웹 푸시. 서버가 양쪽 다 발송하게 이미 구현됨.

### C. 회원 알림 설정 단순화 (형님 최우선 관심사)
- **그냥 알림 받기 = 이미 버튼 하나("허용") 한 번이면 됨.** 앞서 복잡했던 설정들은 "배너·점등" 고급 기능 때문이지 기본 수신엔 불필요.
- 버튼으로 못 하는 것: ①아이폰 "홈 화면에 추가"(애플 제약, 1회 수동) ②안드 배너·점등을 설정 없이 자동 → **네이티브 앱이 자동 세팅**(앱의 장점).
- 선착순 공정성: 화면 점등 대신 **"신청 시작 시각 미리 공지 + 앱 내 카운트다운"** 으로도 해결 가능.
- 더 제대로 가려면 **Capacitor**(웹 코드 유지 + 네이티브 푸시 플러그인)가 정석. 현재 TWA 땜질보다 깔끔.
