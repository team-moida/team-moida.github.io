## 4. 🔴 미해결 (사무실에서 이어갈 것): 카카오 로그인 에러
### 증상
- 카카오 로그인 시 `kauth.kakao.com/err` "일시적인 네트워크 오류" 화면.
- 이성훈(매니저)·정현섭(회장) **둘 다** 동일 → 특정 계정 문제 아님.

### 오늘 확정된 것 (설정/코드 모두 정상)
- 카카오에 **redirect_uri 등록됨** (새/옛 주소 다) — kauth authorize 직접 호출로 검증, 로그인화면으로 정상 이동.
- 카카오 **사이트 도메인 등록됨** — ka(origin) 포함 호출로 검증, 통과.
- 카카오 **SDK 정상**(index.html, v2.7.4), authorize 호출 코드 정상(`handlers-kakao.js`).
- redirect_uri는 `getKakaoRedirectUri()` = location.origin+pathname(index.html 제거) → 새 주소 자동대응.
- 즉 **앱/카카오 설정 잘못 아님.** (어제 6/19 기록에도 "카카오 새 주소 등록 완료"라고 적혀 있음.)

### 남은 원인 후보 (클라이언트 환경)
1. **카카오톡 인앱 브라우저로 열기** — 가장 흔함. 카톡으로 링크 받아 카톡 안에서 열면 카카오 로그인이 잘 막힘.
2. PWA(홈화면 아이콘) standalone 환경에서의 OAuth/Referer 문제.
3. 카카오 측 일시 장애 가능성.

### 다음 단계 (사무실에서)
1. **원인 확정 테스트**: 폰에서 카카오톡 말고 **크롬**으로 `team-moida.github.io` 직접 접속 → 카카오 로그인.
   - 크롬에서 되면 → 인앱 브라우저 문제 확정.
   - 크롬에서도 안 되면 → 카카오 일시장애 의심(시간 두고 재시도) + PC에서도 확인.
2. **인앱 브라우저 문제로 확정 시 코드 해결**: 인앱 브라우저(User-Agent에 KAKAOTALK 등) 감지 →
   외부 브라우저로 자동 유도. 안드로이드는 `intent://...#Intent;...end` 또는
   `kakaotalk://web/openExternal?url=...`. 진입 페이지 `index.html` / `signup-consent.html`.
   → 회원 전체가 다시는 이 에러 안 보도록 영구 방어.
