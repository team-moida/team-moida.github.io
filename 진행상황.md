## 중요 결정 / 사실 (반복 질문 방지)
- **카카오 단톡방 자동 전송 = 불가능** (카카오가 그룹/오픈채팅 전송 API를 안 줌). 공유 버튼으로 수동만 가능.
- **안드로이드 웹 푸시 헤드업 배너 = 불가능** (웹앱은 알림 채널 중요도를 '긴급'으로 못 올림). 네이티브 앱만 가능.
- **선착순 공정성 해법**: 알림 배너에 의존하지 말고 → 신청 시작 시각을 미리 공지 + 인앱 카운트다운(서버시간 동기화) + 자동 푸시. 모두가 시작 시각을 알면 안드로이드 배너 약점이 무의미해짐.
- ★ **"이번 모임 참여자" = weekly_session 기준** (= 관리자가 출석 탭에서 직접 선정한 '선정 명단', member.html에선 `attend.attendActiveParticipants`). registrations(회원 자가 신청)와는 별개. 우리 운영은 주로 직접 선정 방식 → 참여자 관련 기능은 항상 weekly_session/선정명단을 봐야 함 (registrations를 보면 빈 명단/엉뚱한 인원이 나옴).
- ★ **안드로이드 알림 헤드업 배너 근본 해결책 = TWA(APK)**: PWA를 그대로 TWA로 감싼 안드로이드 APK 배포. 아이폰=PWA / 안드로이드=APK, 코드는 하나(웹 고치면 양쪽 반영). APK는 플레이스토어 등록 없이 sideload 배포 예정(서명키 보관 필수).

## 안드로이드 APK 계획 (TWA)

1단계 검증: 테스트 APK 1개 만들어 실제 안드로이드 폰에서 알림 배너 뜨는지 먼저 확인 (여기서 효과 없으면 중단).
2단계: manifest.json 앱 준비 점검 (아이콘 192/512 등).
3단계: PWABuilder로 APK + 서명키 생성, 서명키 백업.
4단계: assetlinks.json을 사이트에 올려 주소창 제거.
5단계: APK 링크/카톡 배포 + 설치 안내문 (안드로이드 출처허용 / 아이폰 홈화면 추가).
6단계(병행): 인앱 카운트다운(선착순 공정성 해법) — 플랫폼 무관하게 든든.

- iOS는 그대로 Safari 웹앱 유지.
